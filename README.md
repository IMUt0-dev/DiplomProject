## Описание проекта

Проект представляет собой полностью документированное и протестированное
ядро информационной системы интернет-магазина электроники. Реализована
реляционная база данных из 10 таблиц с поддержкой:

1. **Гибкого каталога** — хранение разнородных технических характеристик
  товаров (процессоры, память, экраны) через гибридный подход:
  фиксированные поля + JSONB.
2. **ACID-транзакций** — целостная обработка заказов с резервированием
  товаров на складах и фиксацией финансовых транзакций.
3. **Многоуровневой безопасности** — 6 ролей, гранулярные привилегии
  на уровне столбцов и Row-Level Security (RLS) в соответствии
  с Федеральным законом № 152-ФЗ «О персональных данных».
4. **Высокой производительности** — система индексов
  и оптимизированные представления для часто используемых запросов.

База данных спроектирована в СУБД **PostgreSQL 16**. И все SQL-запросы для её создания были собраны в одном файле "SQL-запрос".

## Назначение и цели

**Основное назначение** — служить надёжным и производительным
фундаментом для веб-приложения интернет-магазина электроники.

**Цели ВКР (достигнуты):**

- Спроектировать и реализовать оптимальную реляционную базу данных
- Обеспечить поддержку ключевых бизнес-процессов (каталог, заказы,
  склад, доставка, оплата)
- Гарантировать целостность данных и высокую производительность
- Реализовать требования регуляторов по защите персональных данных

**Возможные сценарии использования:**

- Прототип для реального интернет-магазина электроники
- Эталонная модель для интеграции с CRM, ERP, BI-системами
- Учебный пример полного цикла проектирования БД

## Проверка работоспособности

### Количество записей в таблицах
```sql
SELECT 'manufacturer', COUNT(*) FROM manufacturer
UNION ALL SELECT 'category', COUNT(*) FROM category
UNION ALL SELECT 'product', COUNT(*) FROM product
UNION ALL SELECT 'warehouse', COUNT(*) FROM warehouse
UNION ALL SELECT 'stock', COUNT(*) FROM stock
UNION ALL SELECT 'customer', COUNT(*) FROM customer
UNION ALL SELECT '"order"', COUNT(*) FROM "order"
UNION ALL SELECT 'orderitem', COUNT(*) FROM orderitem
UNION ALL SELECT 'payment', COUNT(*) FROM payment
UNION ALL SELECT 'delivery', COUNT(*) FROM delivery;
```

Ожидаемый результат:

- manufacturer =    10
- category     =    50
- product      =  1000
- warehouse    =     3
- stock        =  3000
- customer     =   500
- "order"      =   600
- orderitem    = ~1800-3000
- payment      =  ~660
- delivery     =  ~600

### Тестирование представлений
```sql
--  Каталог товаров
SELECT * FROM v_product_catalog LIMIT 10;

-- Товары с низким остатком
SELECT * FROM v_low_stock;

```

### Тестирование безопасности

```sql
-- Проверка: анонимный пользователь видит только каталог
SET ROLE role_anonymous;
SELECT * FROM v_product_catalog LIMIT 5;    -- OK
SELECT * FROM customer;                     -- ERROR: permission denied

-- Проверка: role_app не видит хэши паролей
SET ROLE role_app;
SELECT first_name, last_name FROM customer;   -- OK
SELECT password_hash FROM customer;           -- ERROR: permission denied

RESET ROLE;
```
## Кейсы тестирования

### Проверка целостности данных

| Тест | Описание | Ожидаемый результат |
|:---|:---|:---|
| **NOT NULL** | Вставка записи без обязательного поля | Ошибка нарушения ограничения |
| **CHECK (price >= 0)** | Вставка товара с отрицательной ценой | Ошибка нарушения ограничения |
| **Уникальность email** | Регистрация дубликата email | Ошибка нарушения уникального индекса |
| **Внешний ключ (ON DELETE RESTRICT)** | Удаление категории с товарами | Ошибка нарушения ссылочной целостности |
| **Внешний ключ (ON DELETE CASCADE)** | Удаление заказа с позициями | Каскадное удаление позиций |

### Проверка ACID-транзакций

| Тест | Описание | Ожидаемый результат |
|:---|:---|:---|
| **Откат транзакции** | Имитация сбоя при оформлении заказа | Все изменения отменены, склад не тронут |
| **Параллельные транзакции** | Два клиента покупают последний товар | Только один получает товар |

### Проверка Row-Level Security

```sql
-- Установка контекста курьера
SET ROLE role_app;
SELECT set_courier_context('CDEK');

-- Курьер видит только свои доставки
SET ROLE role_courier;
SELECT * FROM delivery;  -- только строки с courier_name='CDEK'
```
